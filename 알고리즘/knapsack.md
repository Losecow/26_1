# 배낭 문제 (The Knapsack Problem)

## 1. 개요 (Introduction)
배낭 문제(Knapsack Problem)는 조합 최적화의 유명한 문제 중 하나입니다. 
도둑이 박물관에 침입하여 가방(Knapsack) 하나에 물건들을 담아갈 때, **배낭이 견딜 수 있는 최대 무게를 초과하지 않으면서 담은 물건들의 총 가치(Benefit/Value)가 최대**가 되도록 담을 물건들을 선택하는 문제입니다.

배낭 문제는 물건을 쪼갤 수 있는지 여부에 따라 두 가지 버전으로 나뉩니다:
1. **0-1 배낭 문제 (0-1 Knapsack Problem)**: 물건을 쪼갤 수 없습니다. (물건을 통째로 챙기거나 아예 챙기지 않아야 함)
2. **분할 가능 배낭 문제 (Fractional Knapsack Problem)**: 물건을 쪼갤 수 있습니다. (물건의 일부분만 담는 것이 가능함)

---

## 2. 분할 가능 배낭 문제 (Fractional Knapsack Problem)
물건을 임의의 비율로 쪼개어 담을 수 있는 경우입니다. 이 문제는 **탐욕 알고리즘(Greedy Strategy)**으로 최적해를 구할 수 있습니다.

### 해결 전략 (Greedy Approach)
1. 각 물건의 **단위 무게당 가치($v_i / w_i$)**를 계산합니다.
2. 단위 무게당 가치가 가장 큰 물건부터 배낭의 한도까지 최대한 많이 담습니다.
3. 배낭에 남은 공간보다 물건의 무게가 무거우면, 배낭의 남은 공간만큼만 물건을 쪼개어 담습니다.
4. 배낭이 꽉 찰 때까지 이 과정을 반복합니다.

---

## 3. 0-1 배낭 문제 (0-1 Knapsack Problem)
물건을 쪼갤 수 없는 경우입니다. 각 물건 $i$는 무게 $w_i$와 가치 $b_i$를 가지며, 배낭의 최대 수용 무게는 $W$입니다. ($w_i$, $b_i$, $W$는 모두 정수)

0-1 배낭 문제는 탐욕 알고리즘(Greedy Approach)으로 최적해를 보장할 수 없으며, 다른 접근법이 필요합니다.

### 3.1 완전 탐색 (Brute Force Approach)
물건이 $n$개일 때, 각 물건을 배낭에 넣거나 넣지 않는 모든 조합을 확인합니다.
- 총 $2^n$개의 조합이 존재합니다.
- 총 무게가 $W$ 이하인 조합 중 가장 가치가 큰 것을 선택합니다.
- **시간 복잡도**: $\Theta(2^n)$ (비효율적이므로 지수 시간이 소요됨)

### 3.2 동적 계획법 (Dynamic Programming)
동적 계획법을 사용하여 부분 문제(Subproblem)를 정의하고 이를 바탕으로 전체 문제를 해결합니다.

#### 부분 문제의 정의 (Flawed vs Correct)
- **잘못된 접근**: 부분 집합 $S_k = \{1, 2, ..., k\}$에 대한 최적해만 고려하면, 이전 단계($S_{k-1}$)의 최적해가 다음 단계($S_k$)의 최적해에 포함되지 않는 경우가 발생합니다.
- **올바른 접근**: 무게 한도를 매개변수 $w$로 추가해야 합니다.
  - **$B[k, w]$**: $1$번부터 $k$번까지의 물건들 중에서 선택하여 총 무게가 $w$를 초과하지 않게 담을 때 얻을 수 있는 최대 가치.

#### 점화식 (Recursive Formula)
아이템 $k$를 배낭에 넣을 수 있는지 여부에 따라 두 가지 경우로 나뉩니다.

1. **$w_k > w$ 인 경우 (물건 $k$가 현재 배낭 용량보다 무거울 때)**
   물건 $k$를 배낭에 넣을 수 없으므로, $k-1$번째 물건까지 고려했을 때의 최적해와 같습니다.
   $$B[k, w] = B[k-1, w]$$

2. **$w_k \le w$ 인 경우 (물건 $k$를 배낭에 넣을 수 있을 때)**
   물건 $k$를 넣지 않은 경우와 넣은 경우 중 더 큰 가치를 선택합니다.
   $$B[k, w] = \max \{ B[k-1, w], \; b_k + B[k-1, w-w_k] \}$$

#### 의사코드 (Pseudocode)
```text
for w = 0 to W
    B[0, w] = 0
for i = 1 to n
    B[i, 0] = 0

for w = 1 to W
    for i = 1 to n
        if w_i <= w  // 물건 i를 넣을 수 있는 경우
            if b_i + B[i-1, w-w_i] > B[i-1, w]
                B[i, w] = b_i + B[i-1, w-w_i]
            else
                B[i, w] = B[i-1, w]
        else         // 물건 i를 넣을 수 없는 경우 (w_i > w)
            B[i, w] = B[i-1, w]
```

### 3.3 분기한정법 (Branch and Bound - Best First Search)
상태 공간 트리(State Space Tree)를 탐색할 때, 유망성(Promising)을 판단하여 가망이 없는 노드는 탐색하지 않고 가지치기(Pruning)하는 방법입니다. 탐색 순서는 **최고 우선 탐색(Best First Search)**을 따릅니다.
* 탐색 전, 물건들은 **단위 무게당 가치($b_i/w_i$) 기준 내림차순**으로 정렬되어 있어야 합니다.

#### 변수 정의
각 노드에서 다음 세 가지 지역 변수와 한 가지 전역 변수를 계산/관리합니다.
- **`benefit`**: 현재 노드까지 선택된 물건들의 가치 합
- **`weight`**: 현재 노드까지 선택된 물건들의 무게 합
- **`bound`**: 현재 노드에서 계속 탐색을 확장했을 때 얻을 수 있는 가치의 **상한선(Upper Bound)**
- **`max_benefit`**: 현재까지 찾은 가장 좋은 해의 가치 (초기값 0)

#### Bound (상한선) 계산 방법
현재 노드 수준이 $i$라고 할 때, 남은 배낭 용량을 최대로 채우기 위해 다음 물건들($i+1, i+2, \dots$)을 순서대로 넣습니다. 배낭 용량 $W$를 초과하기 직전($k-1$번째 물건)까지 통째로 넣고, 남은 용량에 대해서는 $k$번째 물건을 쪼개서 넣는다고 가정(Fractional Knapsack 방식)하여 최대 기대 가치를 계산합니다. 이 값이 `bound`가 됩니다.
$$ \text{bound} = \text{benefit} + \sum_{j=i+1}^{k-1} b_j + (W - \text{tot\_weight}) \times \frac{b_k}{w_k} $$

#### 유망성 검사 (Promising vs Non-promising)
노드를 확장하기 전에 해당 노드가 유망한지 평가합니다.

- **유망하지 않은 경우 (Non-promising)**: 탐색 중단 (가지치기)
  1. `weight > W`: 배낭 용량을 초과한 경우.
  2. `bound <= max_benefit`: 최적의 상황(bound)을 가정해도 현재까지 찾은 최고치(`max_benefit`)보다 작거나 같은 경우.

- **유망한 경우 (Promising)**: 탐색 진행
  - `weight <= W` 이고 `bound > max_benefit` 인 경우.
  - 현재 노드 방문 시 `weight <= W`라면, `max_benefit = max(max_benefit, benefit)`으로 전역 최고값을 업데이트합니다.

**탐색 진행 방식**:
유망한 노드들이 여러 개 있다면, 그 중 `bound` 값이 가장 큰 노드를 우선적으로 선택하여 탐색(Best First Search)을 이어갑니다.
