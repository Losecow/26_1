# 알고리즘 분석: Chapter 15 동적 계획법 (Dynamic Programming)

## 1. 동적 계획법 개요 (Dynamic Programming)
* **정의**: 특정 알고리즘이라기보다는 알고리즘을 설계하는 **설계 패러다임(Design Paradigm)**입니다.
* **기존 패러다임과의 비교**: 분할 정복(Divide and Conquer), 점진적(Incremental), 백트래킹(Backtracking) 등에 이어 새롭게 다루는 기법입니다.
* **사용 목적**: 주로 **최적화 문제(Optimization Problems)**를 해결하는 데 사용됩니다.
  * 최적의 값을 갖는 해답(최솟값 또는 최댓값)을 찾는 것이 목표입니다.

## 2. 최적화 문제란? (Optimization = Making Choices)
* **최적화**: 여러 선택지 중에서 어떤 것이 '최적의 선택'인지 결정하는 과정입니다.
* 우리가 내리는 선택에 따라 풀어야 할 **하위 문제(Subproblems)**가 결정됩니다.
* 어떤 선택이 최적인지 미리 알 수 없으므로, 모든 가능성을 시도해보고 이미 알려진 하위 문제들의 비용을 바탕으로 각 선택의 품질을 평가합니다.
* **예시**:
  * **최단 경로(Shortest-path)**: 포항에서 서울로 가는 여러 경로 중 최단, 최고속, 또는 가장 경치가 좋은 경로 찾기.
  * **작업 스케줄링(Job scheduling)**: 여러 프로세서에 작업을 할당할 때, 최소 시간 내에 최대 작업량을 완료하는 스케줄링 찾기.

## 3. 동적 계획법의 원리와 분할 정복과의 비교
* **원리**: 주어진 문제 $S$의 해를 작고 단순한 동일한 유형의 하위 문제들($S_1, S_2, \dots, S_m$)의 해를 동적으로 결합하여 구성합니다. ($S = combine(S_1, S_2, \dots, S_m)$)
* 가장 작은 문제의 해답부터 시작하여 **상향식(Bottom-up)**으로 전체 문제의 해를 구축합니다.
* **분할 정복(Divide and Conquer) vs 동적 계획법 (DP)**:
  * **분할 정복**: 공통된 하위 문제를 반복적으로 해결하여 불필요한 연산이 많습니다. (예: 단순 피보나치 재귀 호출 시 동일한 값을 계속 다시 계산)
  * **동적 계획법**: 각 하위 문제를 **단 한 번만** 풀고 그 결과를 테이블에 저장해 둡니다. 동일한 하위 문제가 다시 등장하면 다시 계산하지 않고 테이블에서 값을 꺼내 쓰므로 재계산을 방지합니다.

## 4. 행렬 연속 곱셈 문제 (Matrix-Chain Multiplication)
* **문제 정의**: 행렬 $A_1, A_2, \dots, A_n$을 곱할 때, 괄호를 어떻게 묶느냐에 따라 총 곱셈(스칼라 곱)의 횟수가 달라집니다. 이를 최소화하는 괄호 묶음 순서를 찾아야 합니다.
* **두 행렬의 곱셈 비용**: $p \times q$ 행렬과 $q \times r$ 행렬을 곱하면 비용은 $p \cdot q \cdot r$이 됩니다.
* **경우의 수**: $n$개의 행렬에 괄호를 치는 방법의 수($P_n$)는 $\Omega(2^n)$으로 지수적으로 증가합니다. 따라서 모든 경우의 수를 다 확인하는 Brute-force 방식은 연산상 불가능합니다.
* 중복 계산(Recalculation)이 효율성을 떨어뜨리는 가장 큰 원인이 됩니다.

## 5. 동적 계획법 적용 조건 (When Can We Apply DP?)
동적 계획법을 적용하려면 최적화 문제가 다음 두 가지 속성을 가져야 합니다.
1. **최적 부분 구조 (Optimal Substructure)**: 어떤 문제의 최적 해(Optimal solution) 안에, 그 문제의 하위 문제들에 대한 최적 해가 포함되어 있는 구조입니다. (예: 전체 행렬 곱이 최적이라면, 분할된 양쪽 행렬들의 곱도 최적으로 구성되어야 함)
2. **부분 문제 반복 (Overlapping Subproblems)**: 재귀적 해결책이 완전히 새로운 하위 문제를 계속 만들어내는 것이 아니라, 특정한 하위 문제들을 반복적으로 다시 풀어야 하는 구조입니다.

* **최적성의 원리 (Principle of Optimality)**: 문제 $S = combine(S_1, S_2, \dots, S_m)$에서 $S$가 최적 해라면, $S_1, S_2, \dots, S_m$ 역시 해당 하위 문제들의 최적 해여야 합니다. 이 원리가 성립할 때 DP가 가장 효과적입니다.

## 6. 메모이제이션 (Memoization: Avoiding Recalculation)
* **메모이제이션**: 재귀를 활용한 하향식(Top-down) 전략을 유지하면서도 동적 계획법 수준의 효율성을 제공하는 기법입니다.
* 하위 문제의 정답을 저장하는 테이블을 유지하여, 이전에 한 번 계산했던 값은 다시 계산하지 않고 테이블(Memo)에서 꺼내어 씁니다. (상향식 접근인 전통적 DP와 대비됨)

## 7. 동적 계획법 알고리즘 개발 4단계 (Development of DP algorithms)

### Step 1. 최적 해의 구조 특징 파악 (Characterize optimal solution structure)
* 최적 부분 구조를 식별합니다. 행렬 곱 $A_i \dots A_j$를 $A_k$ 기준으로 나눌 때, 최적의 해답은 $A_i \dots A_k$ 부분과 $A_{k+1} \dots A_j$ 부분의 최적 해답을 결합한 것임을 알아냅니다.

### Step 2. 최적 해의 값을 재귀적으로 정의 (Recursively define the value)
* $m[i, j]$ : $A_i$ 부터 $A_j$ 까지 행렬을 곱하는 데 필요한 최소 스칼라 곱셈 횟수.
* $s[i, j]$ : 최적의 괄호 쪼개기 위치인 인덱스 $k$를 저장하는 테이블.
* **점화식**:
  * $i = j$ 일 때: $m[i, j] = 0$
  * $i < j$ 일 때: $m[i, j] = \min_{i \le k < j} \{ m[i, k] + m[k+1, j] + p_{i-1}p_kp_j \}$

### Step 3. 상향식으로 최적 해의 값 계산 (Compute the value bottom-up)
* 크기 정보 배열 $p$, 계산된 최소 비용 테이블 $m$, 분할 지점 테이블 $s$를 준비합니다.
* 부분 문제의 개수는 $\Theta(n^2)$ 입니다.
* `MATRIX-CHAIN-ORDER` 알고리즘을 사용해 체인 길이(r)를 2부터 n까지 순차적으로 늘려가며 $m[i, j]$와 $s[i, j]$ 테이블을 상향식으로 채워 넣습니다.

### Step 4. 계산된 정보로 최적 해 구축 (Construct an optimal solution)
* 이전에 채워 둔 $s[i, j]$ 테이블을 활용하여 실제 최적의 괄호 결합 형태를 알아냅니다.
* `PRINT-OPTIMAL-PARENS(s, i, j)` 라는 재귀 함수를 이용해 $s[i,j]$에 저장된 위치를 바탕으로 좌항과 우항으로 나누며 `(`, `)` 괄호를 적절히 출력합니다.
