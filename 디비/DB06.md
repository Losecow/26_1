## 1. Agenda & SQL Commands

- **SQL DDL (Data Definition Language)**

SQL 명령어는 크게 **DML**과 **DDL**로 나눌 수 있습니다.
- **DML (Data Manipulation Language)**: `SELECT`, `INSERT`, `UPDATE`, `DELETE`
- **DDL (Data Definition Language)**: `CREATE`, `ALTER`, `DROP`

---

## 2. Data Definition Language (DDL)

DDL은 릴레이션(테이블)에 대한 정보를 명시할 수 있게 해줍니다:
- 각 릴레이션의 **스키마 (Schema)**
- 각 속성에 연결되는 **값의 타입 (Domain Types)**
- **무결성 제약조건 (Integrity constraints)**
- 각 릴레이션을 위해 유지해야 할 인덱스 집합
- 각 릴레이션에 대한 보안 및 권한 정보
- 디스크 상의 물리적 저장 구조

## 3. Domain Types in SQL (데이터 타입)

### 3.1 문자열 타입 (Character Strings)
- **`CHAR(n)`**: 고정 길이 문자열 (최대 길이 $n \in [0, 255]$)
- **`VARCHAR(n)`**: 가변 길이 문자열 (최대 길이 $n \in [0, 65535]$)
  - 항상 길이가 같다면 `CHAR`를, 길이가 제각각이라면 `VARCHAR`를 쓰는 것이 좋습니다.
- **`TEXT`**: `VARCHAR` 범위보다 긴 문자열
  - `TINYTEXT` (0 ~ 255 bytes), `TEXT` (0 ~ 65KB), `MEDIUMTEXT` (0 ~ 16MB), `LONGTEXT` (0 ~ 4GB)

> **`CHAR(4)` vs `VARCHAR(4)` 저장 공간 비교**
> - `''` 저장: CHAR는 4바이트(`'    '`), VARCHAR는 1바이트(`''`)
> - `'ab'` 저장: CHAR는 4바이트(`'ab  '`), VARCHAR는 3바이트(`'ab'`)
> - `'abcd'` 저장: CHAR는 4바이트, VARCHAR는 5바이트 (VARCHAR는 길이를 저장하는 오버헤드가 있음)

### 3.2 정수 타입 (Integer)
- **`INT` / `INTEGER`**: 정수 (보통 4 bytes)
- **`SMALLINT`**: 작은 정수 (보통 2 bytes)
- **`BIGINT`**: 큰 정수 (보통 8 bytes)
- `TINYINT` (1 byte), `MEDIUMINT` (3 bytes)도 사용 가능 (RDBMS마다 지원 여부 다름)
  - *참고: Oracle은 오직 `NUMBER` 데이터 타입만 가짐.*

### 3.3 실수 및 소수 타입 (Numeric & Floating Point)
- **`NUMERIC(p,d)`**: 고정 소수점 (정확한 값). 전체 정밀도 `p`자리, 소수점 아래 `d`자리.
  - 예: `NUMERIC(3,1)`은 44.5는 정확히 저장하나, 444.5나 0.32는 불가.
  - MySQL에서 `DECIMAL`은 `NUMERIC`과 동일합니다.
- **`FLOAT`**: 단정밀도 부동 소수점 (근사값)
- **`REAL` / `DOUBLE`**: 배정밀도 부동 소수점 (근사값)

> **`DECIMAL` vs `FLOAT`/`DOUBLE`**
> - `FLOAT`와 `DOUBLE`이 더 빠르지만, **`DECIMAL` 값은 정확합니다.**
> - `SUM()`과 같은 집계 연산 시 `FLOAT`는 미세한 오차(`3.3000000715255737`)가 생길 수 있으나, `DECIMAL(3,2)`는 정확(`3.30`)합니다.

### 3.4 날짜 및 시간 타입 (Date & Time)
- **`DATE`**: `'YYYY-MM-DD'` (예: `'2020-03-01'`)
- **`TIME`**: `'HH:MM:SS'` (예: `'14:30:03.5'`)
- **`DATETIME`**: `'YYYY-MM-DD HH:MM:SS'`
- **`YEAR`**: `'YYYY'` (1901 ~ 2155, 또는 0000)
- **`TIMESTAMP(n)`**: 유닉스 시간 (1970년 1월 1일 이후의 시간). 
  - 보통 시스템 이벤트 로깅에 사용됨. 
  - `n` 크기에 따라 표시 형식이 다름. (예: `TIMESTAMP(14)` = `YYYYMMDDHHMMSS`)

### 3.5 이진 타입 (Binary)
- **`BINARY(n)`**: 고정 길이 이진 바이트 데이터 (문자열이 아님)
- **`VARBINARY(n)`**: 가변 길이 이진 바이트 데이터
- **`BLOB`** (Binary Large OBject): `TINYBLOB`, `BLOB` (65KB), `MEDIUMBLOB` (16MB), `LONGBLOB` (4GB)

## 4. `CREATE TABLE` Construct

새로운 릴레이션(테이블)은 `CREATE TABLE` 명령어로 정의합니다.

```sql
CREATE TABLE r
(
  A1 D1, 
  A2 D2, 
  ..., 
  An Dn,
  (integrity-constraint1),
  ...,
  (integrity-constraintk)
)
```
- `r`: 테이블 이름
- `Ai`: 속성(칼럼) 이름
- `Di`: 데이터 타입

**예제:**
```sql
CREATE TABLE instructor(
  ID          CHAR(5),
  name        VARCHAR(20),
  dept_name   VARCHAR(20),
  salary      NUMERIC(8,2)
);
```

## 5. Integrity Constraints (무결성 제약조건)

SQL은 무결성 제약조건을 위반하는 데이터베이스 업데이트를 방지하여 의미 있는 데이터만 유지되도록 합니다.

### 제약조건의 종류
- **`PRIMARY KEY (A1, ..., An)`**: 기본 키
- **`FOREIGN KEY (Am, ..., An) REFERENCES r`**: 외래 키
- **`UNIQUE`**: 고유 키
- **`NOT NULL`**: Null 값 허용 안 함

### 5.1 `PRIMARY KEY` vs `UNIQUE`
| 비교 항목 | PRIMARY KEY | UNIQUE |
|-----------|------------|--------|
| **용도** | 테이블의 각 행을 고유하게 식별 | PK가 아닌 행을 고유하게 결정 |
| **NULL 허용** | 불가능 | 허용 (DBMS에 따라 하나만 허용하기도 함) |
| **개수** | 테이블당 **단 하나**만 존재 | 테이블당 여러 개 존재 가능 |
| **인덱스** | Clustered index | Non-clustered index |

> **의미:** 두 키 모두 선언된 속성 리스트에서 **중복 값을 허용하지 않는다**는 뜻입니다.

## 6. `CREATE TABLE` 예제 모음

### 6.1 `student` 테이블
```sql
CREATE TABLE student (
  ID          VARCHAR(5),
  name        VARCHAR(20) NOT NULL,
  dept_name   VARCHAR(20),
  tot_cred    NUMERIC(3,0),
  PRIMARY KEY (ID),
  FOREIGN KEY (dept_name) REFERENCES department
);
```
> *(위와 같이 `PRIMARY KEY (ID)`를 따로 빼서 쓰거나, 속성 옆에 `ID VARCHAR(5) PRIMARY KEY`로 바로 적을 수도 있습니다.)*

### 6.2 `takes` 테이블 (복합 키 사용)
```sql
CREATE TABLE takes (
  ID          VARCHAR(5),
  course_id   VARCHAR(8),
  sec_id      VARCHAR(8),
  semester    VARCHAR(6),
  year        NUMERIC(4,0),
  grade       VARCHAR(2),
  PRIMARY KEY (ID, course_id, sec_id, semester, year),
  FOREIGN KEY (ID) REFERENCES student,
  FOREIGN KEY (course_id, sec_id, semester, year) REFERENCES section
);
```

### 6.3 `course` 테이블 (`DEFAULT` 활용)
```sql
CREATE TABLE course (
  course_id   VARCHAR(8),
  title       VARCHAR(50),
  dept_name   VARCHAR(20) DEFAULT 'Comp. Sci',
  credits     NUMERIC(2,0),
  PRIMARY KEY (course_id),
  FOREIGN KEY (dept_name) REFERENCES department
);
```

### 6.4 `neighbors` 테이블 (`NOT NULL` 위반 시 에러)
```sql
CREATE TABLE neighbors(
  name   CHAR(30) PRIMARY KEY,
  addr   CHAR(50) DEFAULT '123 Sesame St.',
  phone  CHAR(16) NOT NULL
);
```
위 테이블에 전화번호를 빼고 데이터를 삽입하려고 하면:
```sql
INSERT INTO neighbors (name) VALUES ('Elmo');
```
👉 `phone` 속성이 **`NOT NULL` 제약조건**이므로 위 삽입 요청은 **거부(Rejected)** 됩니다.

## 7. Table Updates (테이블 및 스키마 업데이트)

### 7.1 튜플(데이터) 업데이트 - DML
- **`INSERT`**: 데이터 삽입
  ```sql
  INSERT INTO instructor VALUES ('10211', 'Smith', 'Biology', 66000);
  ```
- **`DELETE`**: 데이터 삭제
  ```sql
  DELETE FROM student;
  ```
  *(위 명령어는 `student` 테이블의 **모든 튜플(데이터)을 삭제**합니다.)*

### 7.2 테이블 스키마 업데이트 - DDL
- **`DROP TABLE r`**: 테이블 `r` 자체를 완전히 삭제
- **`ALTER TABLE r ADD A D`**: 새로운 속성 `A`(타입 `D`)를 추가
  - 기존에 있던 튜플들은 이 새 속성 자리에 일괄적으로 `Null`이 들어갑니다.
- **`ALTER TABLE r DROP A`**: 속성 `A`를 삭제
  - (속성 삭제는 많은 데이터베이스에서 지원하지 않으나, **MySQL은 지원**합니다.)

**예제:**
```sql
DROP TABLE time_slot_backup;

ALTER TABLE time_slot_backup ADD remark VARCHAR(20);

ALTER TABLE time_slot_backup DROP remark;
```
