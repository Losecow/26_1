# GitHub 활용 및 협업

## GitHub 특징
- Git을 사용하는 프로젝트를 위한 원격 저장소 및 웹 호스팅 서비스.
- 다양한 오픈소스 제공 및 강력한 협업 기능.

## 협업 과정: Fork & Pull Request
- **Fork**: 다른 사람의 원격 저장소를 내 GitHub 계정으로 복사해오는 작업.
- **Pull Request (PR)**: Fork한 저장소에서 수정한 내용을 원본 원격 저장소에 반영해달라고 요청하는 과정.
- **Contributor(기여자) 활동 순서**:
  1. 원본 저장소 Fork.
  2. Fork한 저장소를 로컬 PC로 `clone`.
  3. 브랜치 생성 후 코드 수정 및 커밋.
  4. 원본 저장소를 remote로 추가하고 최신 상태를 `pull`.
  5. 자신의 Fork된 원격 저장소로 `push`.
  6. GitHub 웹사이트에서 원본 저장소로 Pull Request 생성.

## GitHub 부가 기능
- **Organization**: 팀/조직 단위로 저장소를 관리하고 권한을 설정.
- **Issues (이슈)**: 버그, 건의사항, 논의점 등을 기록하고 추적.
- **Wiki**: 프로젝트에 대한 매뉴얼, 구조 등을 마크다운(Markdown) 문서로 작성하여 공유.
- **GitHub Pages**: 정적 웹사이트(HTML, CSS, JS, Markdown 등) 호스팅 서비스 (예: GitHub 블로그). Jekyll 등의 테마를 활용하여 쉽게 블로그 구축 가능.

## Markdown (마크다운)
- 텍스트 기반의 가벼운 마크업 언어.
- 헤더, 목록, 강조, 링크, 이미지, 표, 코드 블록 등 문서를 구조적으로 표현하는 문법 지원.
- `.md` 확장자 사용, GitHub의 README 및 Wiki에서 주로 사용됨.
