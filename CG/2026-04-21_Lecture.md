# 컴퓨터 그래픽스 강의 요약 (2026. 4. 21)

강의 녹음본을 바탕으로 정리한 컴퓨터 그래픽스 핵심 개념 요약입니다.

## 1. 변환 (Transforms)

- **원근 투영 (Perspective Projection)**
  - 평행 투영과 달리 변환 행렬의 마지막 행이 `0, 0, 0, 1`이 아닙니다.
  - 원근 투영에서는 평행선이 더 이상 평행하게 유지되지 않으며, x, y 좌표가 같더라도 z값이 다르면 투영되는 점의 위치도 달라집니다.
- **회전 및 크기 변환 (Rotation & Scaling)**
  - **임의의 점 기준 회전:** 객체를 원점으로 이동 $\to$ 회전 $\to$ 원래 위치로 복귀
  - **임의의 축 기준 회전:** 임의의 축을 표준 축(Standard axis)에 맞춤 $\to$ 회전 $\to$ 원래대로 복귀
  - **특정 방향 기준 크기 변환:** 해당 방향을 표준 축에 맞춤 $\to$ 크기 변환 $\to$ 원래대로 복귀
- **변환 순서:** 일반적으로 **TRS (Scaling $\to$ Rotation $\to$ Translation)** 순서로 적용합니다.
- **행렬식 (Determinant):** 어떤 변환 행렬의 행렬식이 1이라는 것은 객체의 부피(Volume)가 보존된다는 것을 의미합니다 (예: 전단 변환, Shearing).
- **동차 좌표계 (Homogeneous Coordinates):** 행렬 곱셈만으로는 이동(Translation) 연산을 구현할 수 없기 때문에 동차 좌표계를 도입하여 행렬로 표현합니다.

## 2. 셰이딩 (Shading)

- **셰이딩(Shading) vs 셰이더(Shader)**
  - **셰이딩:** 빛과 표면 사이의 상호작용을 계산하여 표면에 닿는 빛의 양을 결정해 사실감을 부여하는 과정입니다.
  - **셰이더:** 버텍스 셰이더나 프래그먼트 셰이더 등 컴퓨터 그래픽스 파이프라인에서 실행되는 작은 프로그램입니다.
- **광원의 종류 (Light Sources)**
  - **방향광 (Directional Light):** 매우 멀리 떨어져 있어 모든 광선이 평행한 빛입니다 (예: 태양).
  - **점광원 (Point Light):** 한 점에서 모든 방향으로 흩어지는 빛으로, 거리가 멀어질수록 세기가 감소합니다.
  - **스포트라이트 (Spotlight):** 특정 방향으로 원뿔 형태로 뻗어가는 빛입니다.
  - **환경광 (Ambient Light):** 수없이 많이 반사되어 출처를 알 수 없는 빛으로, 씬 전체에 균일한 평균 밝기를 제공합니다.
- **빛의 반사 (Reflections)**
  - **정반사 (Specular Reflection):** 빛이 들어온 각도 그대로 반사됩니다. 보는 방향(Viewing direction)에 따라 객체의 외형이 다르게 보입니다 (예: 물, 금속, 플라스틱).
  - **난반사 (Diffuse Reflection):** 빛이 여러 방향으로 고르게 흩어집니다. 어느 방향에서 보든 표면의 모습이 동일합니다 (예: 분필, 천, 석고).
- **BRDF (양방향 반사 분포 함수)**
  - 입사광의 방향과 관찰 방향을 입력으로 받아 반사되는 빛의 양을 반환하는 물리 기반 함수로, 에너지를 보존하고 더욱 정확한 표현이 가능합니다.
  - **램버시안 모델 (Lambertian Model):** 순수한 난반사를 표현하는 단순화된 BRDF로, 관찰 방향을 고려할 필요가 없어 상숫값을 가집니다.
- **셰이딩 기법 (Shading Models)**
  - 법선 벡터(Normal Vector)와 빛의 방향 사이의 각도가 셰이딩의 핵심 요소입니다.
  - **플랫 셰이딩 (Flat Shading):** 각 프리미티브(삼각형)당 하나의 법선 벡터를 사용하여 전체 면의 색상을 동일하게 계산합니다.
  - **고로 셰이딩 (Gouraud Shading):** 버텍스 셰이더에서 각 정점(Vertex)의 빛의 양을 계산한 후, 픽셀로 넘어갈 때 이를 보간(Interpolate)합니다.
  - **퐁 셰이딩 (Phong Shading):** 정점의 법선 벡터를 픽셀 단위로 보간한 뒤, 각 픽셀(Fragment)마다 빛의 양을 계산합니다. 연산량은 가장 많지만 가장 고품질의 결과물을 보여줍니다.

## 3. 텍스처 (Texture)

- **텍스처 맵핑의 4단계**
  1. **Projector Function (투영 함수):** 3D 위치를 텍스처 좌표(UV 좌표)로 투영합니다 (구면, 원통형, 평면 투영 등 활용).
  2. **Corresponder Function (대응 함수):** 보통 0~1 사이의 UV 좌표에 텍스처의 너비/높이를 곱해 실제 위치를 계산합니다. 타일링처럼 1을 넘거나 0보다 작은 UV 범위에 대한 처리 규칙도 여기서 담당합니다.
  3. **Obtain Value (값 참조):** 계산된 위치를 통해 텍스처 맵에서 실제 값을 가져옵니다.
  4. **Value Transform (값 변환):** 빛의 양과 텍스처 값을 혼합하여 최종 색상을 결정합니다.
- **텍셀(Texel)과 픽셀(Pixel):**
  - **텍셀(Texel):** 텍스처(Texture)를 구성하는 최소 단위.
  - **픽셀(Pixel):** 최종 화면(Screen)이나 이미지를 구성하는 최소 단위.
- **에일리어싱 (Aliasing)과 해결책**
  - **원인:** 불충분한 샘플링(Insufficient sampling)으로 인해 발생하며 주로 패턴이 깨지거나 지글거리는(Shimmering) 현상을 보입니다.
  - **해결책 1 (안티에일리어싱 필터):** 입력 데이터를 블러링(Blurring)하거나 스무딩하여 값의 급격한 변화를 줄입니다.
  - **해결책 2 (밉맵핑, Mip-mapping):** 거리에 따라 다양한 해상도로 미리 축소해둔 텍스처 집합을 만들어두고 적절하게 골라 사용하여 근본적인 에일리어싱을 줄입니다.
- **범프 맵핑 (Bump Mapping):**
  - 모델의 지오메트리를 실제로 수정하지 않고, **법선(Normal) 정보가 담긴 텍스처**를 사용하여 표면의 복잡한 굴곡과 디테일을 시각적으로 구현하는 기법입니다.
