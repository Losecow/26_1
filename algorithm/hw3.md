# Homework 3: 유리구슬 떨어뜨리기 (Glass Ball Dropping Problem)

## 1. 문제 설명 (Problem Description)

K개의 유리구슬과 N층짜리 건물이 주어집니다. 이 건물에는 다음과 같은 조건을 만족하는 특정 층 $H$ (임계 층)가 존재합니다.

- 임의의 $H$층 이하에서 구슬을 떨어뜨리면 구슬은 **깨지지 않습니다**.
- 임의의 $H$층 초과 층에서 구슬을 떨어뜨리면 구슬은 **깨집니다**.

우리의 목표는 $H$의 정확한 값을 직접 계산하는 것이 아니라, $H$를 알아내기 위해 보장되어야 하는 최악의 경우(worst-case)에서 **최소 몇 번의 시도(drop)가 필요한지**를 알아내는 것입니다.

### 주요 제약 조건:
- 구슬이 깨지면, 그 구슬은 다시 사용할 수 없습니다.
- 구슬이 깨지지 않으면, 다음 시도에서 다시 사용할 수 있습니다.
- 반드시 최악의 경우를 상정하고 $H$의 값을 찾아낼 수 있음을 보장해야 합니다.
- 목적은 이 최악의 경우에 발생할 수 있는 시도 횟수를 **최소화**하는 것입니다.

### 예시 (Examples)

#### Example 1
- **Input:** $K = 1$ (유리구슬 1개), $N = 10$ (10층)
- **Output:** 10
- **Explanation:** 유리구슬이 하나밖에 없으므로 1층부터 10층까지 차례대로 하나씩 확인해야 합니다. 따라서 최악의 경우 $H$를 찾기 위해 10번의 시도가 필요합니다.

#### Example 2
- **Input:** $K = 2$ (유리구슬 2개), $N = 6$ (6층)
- **Output:** 3
- **Explanation:** 최적의 층을 선택하여 탐색을 최적화할 수 있습니다.
  1. **첫 번째 시도 (3층에서 떨어뜨림):**
     - **만약 깨진다면:** $H$는 3 미만이므로 1층을 확인합니다.
       - 1층에서 깨질 경우 ($H=0$): 총 2번의 시도로 완료.
       - 1층에서 깨지지 않을 경우: 2층을 확인합니다. 2층에서 깨지든($H=1$) 안 깨지든($H=2$) 최대 3번의 시도로 완료.
     - **만약 깨지지 않는다면:** $H$는 최소 3 이상이므로 남은 층 {4, 5, 6}을 확인하기 위해 5층으로 이동합니다.
  2. **두 번째 시도 (5층에서 떨어뜨림):**
     - **만약 깨진다면:** $H$는 5 미만이므로 4층만 확인하면 됩니다. 4층에서 깨질 경우($H=3$)와 안 깨질 경우($H=4$) 모두 최대 3번의 시도로 완료.
     - **만약 깨지지 않는다면:** $H$는 최소 5 이상이므로 6층을 확인합니다. 6층에서 깨지든($H=5$) 안 깨지든($H=6$) 최대 3번의 시도로 완료.
  
  이러한 접근 방식을 따르면, 최악의 경우에도 단 3번의 시도만 필요합니다.

#### Example 3
- **Input:** $K = 3$ (유리구슬 3개), $N = 14$ (14층)
- **Output:** 4
- **Explanation:** 최적의 전략을 사용했을 때, 최악의 경우 필요한 최소 시도 횟수는 4번입니다.

## 2. 해결 방법 및 알고리즘 설명

이 문제는 구슬이 깨질 때와 깨지지 않을 때, 두 가지의 갈림길로 나뉘며 각각 다른 상태가 되기 때문에, 문제를 더 작은 하위 문제로 쪼개어 접근할 수 있는 **동적 계획법(Dynamic Programming)** 혹은 **재귀(Recursion)** 방식으로 해결할 수 있습니다.

### 점화식 도출 (Recurrence Relation)

$E(K, N)$ 을 $K$개의 구슬과 $N$층의 건물이 있을 때, $H$를 찾기 위해 필요한 최소 시도 횟수라고 정의합니다. 

#### [기본 조건 (Base Cases)]
1. **구슬이 1개만 남은 경우 ($K = 1$)**
   - 유일한 구슬이 깨지면 안 되므로, 반드시 1층부터 $N$층까지 차례대로 확인해야 합니다.
   - 따라서 $E(1, N) = N$ 이 됩니다.
2. **건물이 1층인 경우 ($N = 1$)**
   - 1번만 떨어뜨려보면 깨지는지 여부를 바로 알 수 있습니다.
   - 따라서 $E(K, 1) = 1$ 이 됩니다.
3. **건물이 0층인 경우 ($N = 0$)**
   - 시도할 층이 없으므로 구슬을 떨어뜨릴 필요가 없습니다.
   - 따라서 $E(K, 0) = 0$ 이 됩니다.

#### [재귀적 조건 (Recursive Case)]
효율적인 $H$를 찾기 위해, 1층부터 $N$층 중 임의의 층 $X$ ($1 \le X \le N$)에서 구슬을 하나 떨어뜨려 본다고 가정합시다. 이때 결과는 두 가지로 나뉩니다.

1. **Case 1: $X$층에서 구슬이 깨진 경우**
   - 임계 층 $H$는 무조건 $X$층보다 아래에 있습니다.
   - 남은 구슬은 $K-1$개이고, 이제 우리가 테스트해야 할 층은 1층부터 $X-1$층까지 총 $X-1$개입니다.
   - 점화식: $E(K-1, X-1)$
2. **Case 2: $X$층에서 구슬이 깨지지 않은 경우**
   - 임계 층 $H$는 $X$층 이상입니다.
   - 구슬은 여전히 $K$개가 그대로 남아 있고, 테스트해야 할 층은 $X+1$층부터 $N$층까지 총 $N-X$개입니다.
   - 점화식: $E(K, N-X)$

우리는 최악의 상황에서도 $H$를 찾아야 하므로(운이 나쁜 경우), 두 가지 경우 중 더 많은 시도를 요구하는 **최댓값(max)**을 택해야 합니다. 그리고 현재 $X$층에서 1번 구슬을 떨어뜨렸으므로 1을 더해줍니다.
이후 우리가 구하고 싶은 것은 어떤 $X$층을 선택해야 전체 시도 횟수를 최소화할 수 있는지이므로, 1부터 $N$까지 모든 $X$에 대해 최솟값을 구합니다.

최종 점화식은 다음과 같습니다.
$$E(K, N) = 1 + \min_{1 \le X \le N} (\max(E(K-1, X-1), E(K, N-X)))$$

## 3. 코드 해설 (C++)

`algorithm/hw3.cpp` 파일에는 위의 점화식을 바탕으로, **재귀 (Recursion)** 방식과 **동적 계획법 (Dynamic Programming)** 방식의 2가지 해결법이 모두 구현되어 있습니다.

### 1) 재귀 (Recursive) 구현
점화식을 그대로 함수 `recursive(int k, int n)` 형태로 옮겼습니다. 
모든 분기를 따라 탐색하므로 시간 복잡도는 지수적($O(2^N)$ 이상)으로 크게 증가합니다. $N$이 커질 경우 동일한 하위 문제를 반복적으로 계산하므로 실행 시간이 기하급수적으로 늘어날 수 있다는 단점이 있습니다.

### 2) 동적 계획법 (DP) 구현
재귀의 성능 저하 원인인 '중복 연산'을 막기 위해, 한 번 계산한 $E(K, N)$의 값을 저장하는 테이블(배열)을 활용합니다.
`vector<vector<int>> dp(k + 1, vector<int>(n + 1, 0))` 테이블을 생성하고, 기저 조건(Base case)을 미리 채운 뒤, Bottom-up 방식으로 구슬 수 $i$와 층 수 $j$를 점점 늘려가며 정답을 도출합니다.
이렇게 하면 삼중 for문을 돌게 되므로 시간 복잡도는 $O(K \times N^2)$으로, 재귀 방식에 비해 매우 빠르게 정답을 찾을 수 있습니다. 또한, VLA (가변 길이 배열) 대신 `std::vector`를 사용해 메모리 안정성도 고려했습니다.

### 3) 성능 비교 (시간 측정)
C++의 `<chrono>` 라이브러리를 활용하여 두 방식의 실행 시간(microseconds 단위)을 측정하여 화면에 출력합니다. $N$과 $K$의 값이 커질수록 재귀 방식과 DP 방식 간의 압도적인 속도 차이를 확인할 수 있습니다.
